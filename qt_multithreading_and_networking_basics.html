C++ / Qt multithreading and networking basics

 Qt is very extensive and powerful framework based on C++ for wide range of applications, it's often recommended answer for questions like "What should I use to create GUI in C++?" or "Is there any good networking library for C++?". Important thing is, that with Qt we can create portable code and it has very good documentation and wide community always ready to help. Qt also offers development environment Qt Creator that is also great and recommended when working with Qt to get fully into this ecosystem.
I made the same decision when I have choosen to work with Qt for GUI design, so I decided to recreate WinHttp functionality with Qt Network library. My plan for this has been already hinted in WinHttp article to design application connecting with Binance cryptocurrency market and below representation is solid draft for fully functional multithread program with network solution.
 Because Qt is fully functional framework for wide range of purposes, its one of main classes is definitely QObject and it's certain that user finally bumps into it, as lots of classes inherit from it. With this subject is stricly connected macro Q_OBJECT, as quoting the documentation:
The Q_OBJECT macro must appear in the private section of a class definition that declares its own signals and slots or that uses other services provided by Qt's meta-object system.
Also QApplication class that we use in main.cpp is not an exception, its classes inheritance path goes like:
QApplication -> QGuiApplication -> QCoreApplication -> QObject.
#include "binRaider.h" #include <QApplication> int main(int argc, char *argv[]) { QApplication a(argc, argv); binRaider w; w.show(); return a.exec(); }
We use QApplication class that provides all the functionality for QWidget-based applications but we just use it as the one to start the program with exec() function and before we use binRaider class which is the main one with show() function - this class inherits from QMainWindow in public mode and it inherits from QWidget - that's why we need to define QWidget pointer argument in constructor. It's all visible in binRaider.h header below:
